## 一、项目概述

一个面向 C 端用户的影视内容平台，**YouTube 暗色主题风格**，支持电影浏览/搜索/收藏、AI 智能客服对话、视频播放（VIP 权限控制）、海报识图找片、用户会员体系、管理后台等完整业务闭环。

| 指标 | 数据 |
|------|------|
| 总提交数 | 28 |
| 前端源码文件 | 73+ |
| 后端源码文件 | 43 |
| 总代码行数 | ~7,200 |
| 数据库电影数 | 100（豆瓣 Top 250） |
| 可用视频数 | 6部真实预告片 + 94部演示视频 |
| 支持角色 | USER / ADMIN |
| 支持设备 | 手机 / 平板 / 桌面 五档断点 |

## 六、API 端点总览

### 认证 `/api/auth`

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| POST | `/api/auth/register` | 注册 | Public |
| POST | `/api/auth/login` | 登录 -> JWT | Public |
| GET | `/api/auth/me` | 当前用户信息 | Login |

### 电影 `/api/movies`

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET | `/api/movies` | 列表 (page/genre/sort/q) | Public |
| GET | `/api/movies/{id}` | 详情 (含 hasVideo/videoUrl) | Public |
| GET | `/api/movies/{id}/video` | 电影关联视频信息 | Public |
| GET | `/api/movies/genres` | 分类列表 | Public |
| GET | `/api/movies/search?q=` | 关键词搜索 | Public |

### 搜索 `/api/search`

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET | `/api/search/suggestions?q=` | 智能联想（全字段匹配，5条） | Public |

### 海报 `/api/posters`

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET | `/api/posters/{movieId}` | 二进制图片 (image/webp) | Public |

| 方法 | 路径 | 说明 | 权限 |

| GET | `/api/watchlist/check/{movieId}` | 检查是否已收藏 | Login |

### AI `/api/ai`

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|

## 七、数据库表

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| `user` | 用户表 | id, username, password(BCrypt), email, role(USER/ADMIN) |
| `movie` | 电影表 | id, title, rating, year, genre, director, poster_url, poster_data(LONGBLOB), **has_video, video_url, required_vip_level** |
| `watchlist` | 片单收藏 | id, user_id(FK), movie_id(FK) |
| `comment` | 评论表 | id, user_id(FK), movie_id(FK), content(TEXT), rating |
| `video` | 视频表 | id, user_id, title, url, cover_url, status(PENDING/APPROVED/REJECTED) |
| `category` | 分类表 | id, name |
| `tag` | 标签表 | id, name |
| `video_tag` | 视频标签关联 | video_id(FK), tag_id(FK) |

> **v1.2.0 新增**: movie 表新增 `has_video`, `video_url`, `required_vip_level` 三个字段

## 九、启动命令

### Quick Start

```
Double-click start-all.bat
```

### Manual Start

```bash
# 1. Docker
docker compose up -d

# 2. Backend (new terminal)
set JAVA_HOME=D:\JDK21
set PATH=%JAVA_HOME%\bin;D:\maven\maven-mvnd-2.0.0-rc-3-windows-amd64\mvn\bin;%PATH%
cd backend
mvn spring-boot:run "-Dspring-boot.run.jvmArguments=-DDEEPSEEK_API_KEY=your-key"

# 3. Frontend (new terminal)
cd frontend
npm install   # first time only
npm run dev
```
