一、需求概述
------

在现有影视平台基础上新增**视频播放核心功能**，实现从上传到播放的完整业务闭环。

### 1.1 核心诉求

表格

| 需求项  | 说明                               |
| ---- | -------------------------------- |
| 视频上传 | 管理员在后台统一上传，填标题 + 简介 + 分类 + 封面    |
| 视频审核 | 管理员上传即发布，无需二次审批                  |
| 视频播放 | 电影详情页内嵌播放器，下方保留评论 / 简介区域         |
| 权限控制 | 视频与 VIP 等级挂钩（USER/VIP/VVIP/SVIP） |
| 标识展示 | 电影卡片显示「可播放」角标 + VIP 等级标签         |
| 格式转换 | 后端 FFmpeg 统一转 HLS，保证跨端兼容         |

### 1.2 设计原则

* **最小改动**：复用现有电影表和前端组件，仅做增量开发
* **流程简化**：管理员上传即发布，无审核队列
* **权限复用**：直接使用现有 VIP 等级体系
* **体验优先**：列表页即可见播放标识

三、数据模型设计

--------

### 3.1 新增表：`video`

sql
    CREATE TABLE video (
        id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '视频ID',
        movie_id BIGINT NOT NULL COMMENT '关联电影ID',
        title VARCHAR(255) NOT NULL COMMENT '视频标题',
        description TEXT COMMENT '视频简介',
        category VARCHAR(50) COMMENT '分类',
        cover_url VARCHAR(500) COMMENT '封面图URL',
        original_filename VARCHAR(255) COMMENT '原始文件名',
        original_path VARCHAR(500) COMMENT '原始文件路径',
        file_size BIGINT COMMENT '文件大小（字节）',
        duration INT COMMENT '视频时长（秒）',
        resolution VARCHAR(20) COMMENT '分辨率',
        hls_path VARCHAR(500) COMMENT 'HLS播放地址（.m3u8路径）',
        hls_segment_dir VARCHAR(500) COMMENT 'HLS切片目录',
        transcode_status VARCHAR(20) DEFAULT 'PENDING' COMMENT 'PENDING/PROCESSING/SUCCESS/FAILED',

        required_vip_level VARCHAR(20) DEFAULT 'USER' COMMENT 'USER/VIP/VVIP/SVIP',
        status VARCHAR(20) DEFAULT 'PUBLISHED' COMMENT 'DRAFT/PUBLISHED/ARCHIVED',
        created_by BIGINT COMMENT '创建人（管理员ID）',
        created_at DATETIME DEFAULT NOW(),
        published_at DATETIME,

        view_count INT DEFAULT 0 COMMENT '播放次数',
        like_count INT DEFAULT 0 COMMENT '点赞数',

        FOREIGN KEY (movie_id) REFERENCES movie(id) ON DELETE CASCADE,
        FOREIGN KEY (created_by) REFERENCES user(id),
        INDEX idx_movie_id (movie_id),
        INDEX idx_status (status),
        INDEX idx_vip_level (required_vip_level)
    ) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

四、种子数据方案

--------

### 4.1 策略说明

3 部真实预告片 + 97 部共用演示视频，确保 100 部电影全部可播放。

表格

| 电影范围      | 视频类型  | 来源                   |
| --------- | ----- | -------------------- |
| 前 3 部     | 真实预告片 | YouTube 官方频道下载（公开合法） |
| 第 4-100 部 | 演示视频  | 同一个 demo.mp4 重复填充    |

### 4.2 推荐预告片来源

表格

| 电影     | YouTube 搜索关键词                             |
| ------ | ----------------------------------------- |
| 肖申克的救赎 | The Shawshank Redemption official trailer |
| 盗梦空间   | Inception official trailer                |
| 星际穿越   | Interstellar official trailer             |

### 4.3 演示视频准备

1. 自己录制或使用开源测试视频（如 Big Buck Bunny）
2. 时长 30 秒～2 分钟，大小≤50MB
3. 命名 demo.mp4，存放于 `/uploads/videos/demo/`

### 4.4 批量导入方式

先手动上传 3 部真实预告片，再执行 SQL 批量关联：

sql
    -- 为剩余97部电影插入同一演示视频
    INSERT INTO video (movie_id, title, description, category, hls_path, required_vip_level, status, created_by, created_at, published_at)
    SELECT 
        m.id, CONCAT(m.title, ' - 正片'), m.description, m.genre,
        '/videos/demo/playlist.m3u8', 'USER', 'PUBLISHED',
        1, NOW(), NOW()
    FROM movie m
    WHERE m.id > 3 AND m.id NOT IN (SELECT DISTINCT movie_id FROM video WHERE movie_id IS NOT NULL);
    -- 更新 movie 关联字段
    UPDATE movie m
    JOIN video v ON m.id = v.movie_id
    SET m.has_video = TRUE, m.video_id = v.id, m.required_vip_level = v.required_vip_level;
五、API 接口设计

----------

### 5.1 管理员接口

表格

| 方法     | 路径                       | 说明           | 权限    |
| ------ | ------------------------ | ------------ | ----- |
| POST   | `/api/admin/videos`      | 上传视频（含文件）    | ADMIN |
| PUT    | `/api/admin/videos/{id}` | 更新视频信息       | ADMIN |
| DELETE | `/api/admin/videos/{id}` | 删除视频（物理删除文件） | ADMIN |
| GET    | `/api/admin/videos`      | 获取视频列表       | ADMIN |
| GET    | `/api/admin/videos/{id}` | 获取视频详情       | ADMIN |

### 5.2 公开接口

表格

| 方法   | 路径                            | 说明               | 权限       |
| ---- | ----------------------------- | ---------------- | -------- |
| GET  | `/api/videos/{id}`            | 获取视频详情（含 HLS 地址） | 按 VIP 等级 |
| GET  | `/api/videos/movie/{movieId}` | 获取电影关联的视频        | 按 VIP 等级 |
| POST | `/api/videos/{id}/view`       | 播放次数 + 1         | PUBLIC   |

### 5.3 关键接口定义

#### POST /api/admin/videos

text
    multipart/form-data:
      file: [视频文件]
      movieId: 123
      title: "肖申克的救赎 正片"
      description: "..."
      category: "剧情"
      requiredVipLevel: "USER"
      coverFile: [封面图片]

#### GET /api/videos/{id} - 权限校验逻辑

1. 获取视频 required_vip_level
2. 获取当前用户 VIP 等级
3. 等级不足返回 403 Forbidden
4. 等级足够返回视频详情（含 HLS 地址）

改动位置: `components/movie/MovieCard.jsx`

### 6.4 电影详情页 - 内嵌播放器

七、技术实现要点

### 7.1 后端技术选型

表格

| 组件    | 选型                         | 说明        |
| ----- | -------------------------- | --------- |
| 视频转码  | FFmpeg（命令行调用）              | 最成熟       |

| 文件存储  | 本地文件系统                     | Demo 环境够用 |
| 流媒体协议 | HLS（m3u8 + ts）             | 浏览器兼容性最好  |

### 7.2 前端技术选型

表格

| 组件   | 选型                    |
| ---- | --------------------- |
| 视频播放 | hls.js                |
| 播放控制 | 原生 `<video>` + hls.js |
| 进度存储 | localStorage          |

### 7.3 FFmpeg 转码命令

bash

运行
    ffmpeg -i input.mp4 \
      -c:v libx264 -preset medium -crf 23 \
      -c:a aac -b:a 128k \
      -hls_time 10 \
      -hls_list_size 0 \
      -hls_segment_filename "output_%03d.ts" \
      output.m3u8
