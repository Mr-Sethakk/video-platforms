# 产品设计说明文档：智能搜索与历史记录功能

**文档版本**：V1.0  
**功能名称**：首页搜索框 - 智能联想 + 搜索历史管理  
**核心用户**：普通用户（找片源）  
**防抖延迟**：700ms  
**历史记录上限**：20条/用户

---

## 1. 搜索框入口与布局

- **固定位置**：首页顶部，居中或靠左放置。
- **基础构成**：搜索输入框 + 搜索按钮（放大镜图标）+ 清空输入按钮（输入内容后出现）。
- **默认态**：获得焦点时，展开下拉面板；失焦时收起下拉面板。

---

## 2. 搜索联想（自动补全）

### 2.1 触发机制

- 用户每输入一个字符，触发**防抖延迟 700ms** 后发起联想请求。
- 若用户在 700ms 内继续输入，重新计时，不重复请求。

### 2.2 联想范围

- **全字段匹配**：同时支持电影名称、导演、演员、上映年份、类型标签。
- 示例输入"诺兰 2020" → 匹配 Christopher Nolan 导演 + 2020 年上映的电影。

### 2.3 联想结果展示

- **最多展示 5 条**。
- 每条展示内容：**关键词高亮匹配部分** + 类型标签（如"电影/导演/演员"）。

### 2.4 联想结果排序规则（优先级从高到低）

| 优先级 | 排序依据 |
| :---: | :--- |
| 1 | **评分**（豆瓣/平台评分，从高到低） |
| 2 | **匹配度**（文本相似度，从高到低） |
| 3 | **热度**（搜索次数/播放量，从高到低） |
| 4 | **上映时间**（从新到旧） |

### 2.5 点击联想词的行为

- 点击联想列表中的任意一项 → **直接跳转**到该电影的**详情页**。
- 不经过中间结果列表页。

---

## 3. 搜索历史

### 3.1 存储内容

每条历史记录保存：
- **完整搜索关键词**（用户实际输入的文本）
- **搜索时间戳**（精确到秒）

不保存用户点击的电影 ID（仅存关键词，便于复用搜索）。

### 3.2 存储上限与去重规则

- 每个用户（按 User ID）独立存储，**最多 20 条**。
- **去重规则**：若用户搜索的关键词与已有记录**完全一致**（不区分大小写），则**合并为一条**，并将该记录的**时间戳更新为最新一次搜索时间**。
- 超出 20 条时，自动删除**时间戳最旧**的一条。

### 3.3 展示位置与排序

- **展示位置**：搜索框获得焦点时，在下拉面板中展示。
  - 有历史记录时，历史区域位于联想结果**上方**，标题为"最近搜索"。
  - 无历史记录时，不展示该区域。
- **排序方式**：按搜索时间**倒序排列**（最近搜索的排在最前面）。

### 3.4 历史记录的删除操作

| 操作类型 | 交互方式 | 确认机制 |
| :--- | :--- | :--- |
| **单条删除** | Web端：鼠标悬停时，条目右侧出现"×"按钮，点击删除。<br>移动端：左滑条目，露出"删除"按钮，点击删除。 | **无需确认**，直接删除该条。 |
| **清空全部** | 列表右上角放置"清空"文字按钮，点击触发。 | **需要二次确认弹窗**：<br>标题："清空所有搜索历史？"<br>按钮："确认清空" / "取消" |

---

## 4. 搜索执行与结果展示

### 4.1 触发搜索的方式

- 用户在输入框中输入内容后，点击**搜索按钮**或按键盘 **Enter 键**。

### 4.2 空关键词处理

- 若输入框内容为**空字符串**或**仅包含空格**，点击搜索/Enter 时：
  - **不发起搜索请求**，直接展示**全部电影列表**（按默认排序，如热度）。

### 4.3 搜索结果的展示方式

- **无论匹配结果数量多少**（0个、1个、多个），统一先进入**中间结果列表页**。
- 结果列表页为独立页面/全屏面板，非下拉浮层。

### 4.4 搜索结果排序规则（优先级从高到低）

| 优先级 | 排序依据 |
| :---: | :--- |
| 1 | **综合匹配度**（文本相似度综合得分） |
| 2 | **热度**（搜索次数/播放量） |
| 3 | **评分**（豆瓣/平台评分） |

### 4.5 无结果状态

- 展示内容（三要素）：
  1. 提示文案："未找到与"{关键词}"相关的电影"
  2. **热门电影推荐列表**（横向滚动或网格，6～8部）
  3. 引导文案："试试换个关键词吧"

---

## 5. 特殊字符与多关键词处理

| 规则项 | 处理方式 |
| :--- | :--- |
| **大小写** | 不区分大小写。"Avengers"与"avengers"视为相同关键词，命中同一条历史记录。 |
| **特殊字符** | 自动过滤 Emoji、中英文标点符号（如 ，。！？,.!?），仅保留**汉字、字母、数字、空格**进行匹配。 |
| **多关键词组合** | 用户输入多个词（如"诺兰 科幻 2020"），系统**仅取第一个空格前的关键词**进行搜索（即只搜"诺兰"），其余词忽略。 |

## 6. 全流程状态与交互反馈

| 状态名称 | 触发条件 | UI交互反馈 | 文案提示 |
| :--- | :--- | :--- | :--- |
| **空闲态** | 搜索框未获得焦点 | 显示"搜索电影、导演、演员..."占位文案 | 无 |
| **输入中（防抖等待）** | 用户停止输入，700ms 倒计时内 | 输入框右侧显示**加载小圆点**（或骨架动画） | 无 |
| **联想结果返回** | 联想请求成功返回 | 下拉面板展开，联想列表从上至下依次淡入 | 无 |
| **联想无结果** | 联想请求返回空 | 下拉面板显示"暂无匹配的影片或影人" | "暂无匹配的影片或影人" |
| **搜索执行中** | 用户点击搜索/Enter | 结果列表页顶部显示加载骨架屏 | 无 |
| **搜索有结果** | 搜索结果返回 ≥1 条 | 列表页以卡片网格/列表展示结果 | 顶部显示"找到 {N} 部相关影片" |
| **搜索无结果** | 搜索结果返回 0 条 | 展示"未找到"插画 + 热门推荐列表 | "未找到与"{关键词}"相关的电影" |

## 7. 边界情况与兜底策略

- **网络异常**：联想请求超时（>3s）或失败时，**静默降级**，不展示联想列表，用户仍可手动点击搜索进入结果页。
- **历史记录为空**：搜索框下拉面板中，不展示"最近搜索"区域，仅显示联想结果。
- **超长关键词**：输入超过 50 个字符时，自动截断前 50 个字符作为搜索关键词，并 Toast 提示"关键词过长，已自动截断"。
- **频繁刷新**：防抖机制已覆盖，700ms 内不重复请求。

## 8. 关键埋点建议

| 埋点事件 | 触发时机 | 携带参数 |
| :--- | :--- | :--- |
| `search_input_start` | 用户点击搜索框获得焦点 | 无 |
| `search_typing` | 用户每次输入/删除字符（防抖前） | `input_length`（当前输入长度） |
| `search_suggestion_show` | 联想结果返回并展示 | `suggestion_count`、`suggestion_list`（前5条ID） |
| `search_suggestion_click` | 用户点击联想列表某一项 | `clicked_item_id`、`clicked_item_type`（电影/导演/演员） |
| `search_execute` | 用户点击搜索按钮或按 Enter | `keyword`（最终搜索词）、`result_count` |
| `search_result_click` | 用户在结果列表页点击某部电影 | `movie_id`、`position_in_list` |
| `search_no_result` | 搜索结果为空 | `keyword` |
| `search_history_delete_one` | 用户删除单条历史记录 | `deleted_keyword` |
| `search_history_clear` | 用户清空全部历史记录 | `remaining_count`（清空前条数） |
