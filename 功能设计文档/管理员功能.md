# 视频平台后台管理系统 — 功能规格文档

> 摘自 CineStream Pro 管理员后台，可直接用于 AI 辅助开发。

---

## 一、系统架构概览

```
后台路径: /             (Layout 布局: 左侧菜单 + 右侧内容区)
登录路径: /login
认证方式: JWT Token (用户名+密码登录)
技术栈:   React 19 + Tailwind CSS 4 + ECharts + Zustand 状态管理
```

---

## 二、页面功能清单（共 8 个管理页面 + 1 个登录页）



### 2. 仪表盘 `/`（首页）

**功能：**

- 4 个统计卡片：今日新增用户、活跃用户、新增订单、今日收入
- 收入趋势折线图（近 7 天，支持动态切换天数）
- 会员分布环形饼图（Free / VIP / SVIP）
- 内容分布横向进度条（Movies / Series / Anime / Variety——各类型内容数量及占比）
- 热门内容 Top 10 排行榜表格（排名、标题、评分、播放量、类型）

**统计卡片数据：**
| 指标 | 说明 |
|------|------|
| todayNewUsers | 今日新增用户数 |
| todayNewUsersChange | 新增用户环比变化(%) |
| activeUsers | 活跃用户数 |
| activeUsersChange | 活跃用户环比变化(%) |
| todayOrders | 今日新增订单数 |
| todayOrdersChange | 订单环比变化(%) |
| todayRevenue | 今日收入(元) |
| todayRevenueChange | 收入环比变化(%) |
| totalMembers | 总会员数 |

**图表数据：**
| 图表 | 说明 |
|------|------|
| 收入趋势 | `[{date, revenue}, ...]` 按天/周聚合 |
| 会员分布 | `[{name: "Free", value: 70}, {name: "VIP", value: 20}, {name: "SVIP", value: 10}]` |
| 内容分布 | `[{name: "Movies", value: 450, max: 520}, ...]` |
| 热门内容 | 按 viewCount 降序取 Top 10 |

### 3. 影视管理 `/movies`

**功能：**

- 列表展示：表格形式，支持分页（每页 20 条）
- 搜索过滤：按标题关键词搜索、按类型筛选（电影/电视剧/综艺/动漫/短剧）、按状态筛选（草稿/已发布/已归档）
- 批量操作：多选后批量发布/批量下架/批量删除
- 新增影视：弹出 Modal 表单，分段填写
- 编辑影视：点击编辑按钮，Modal 回填已有数据
- 删除影视：确认后删除
- 列表展示字段：海报缩略图、标题、类型、评分（黄色星星）、播放量、状态标签（彩色badge）

**新增/编辑 Modal 表单 — 三段式结构：**

#### 段一：基本信息

| 字段            | 类型       | 必填  | 说明                                                      |
| ------------- | -------- | --- | ------------------------------------------------------- |
| title         | text     | 是   | 标题                                                      |
| originalTitle | text     | 否   | 原文标题/英文名                                                |
| type          | select   | 是   | 类型：movie=电影 / tv=电视剧 / variety=综艺 / anime=动漫 / short=短剧 |
| status        | select   | 是   | 状态：draft=草稿 / published=已发布 / archived=已归档              |
| quality       | select   | 否   | 画质：SD / HD / FHD / 4K                                   |
| releaseDate   | date     | 否   | 上映日期                                                    |
| rating        | number   | 否   | 评分 0-10，步进0.1                                           |
| duration      | number   | 否   | 时长（分钟）                                                  |
| overview      | textarea | 否   | 剧情简介                                                    |

#### 段二：分类关联

| 字段   | 说明                                                       |
| ---- | -------------------------------------------------------- |
| 类型分类 | 多选按钮组，点击切换（Action/Comedy/Drama/Sci-Fi/Horror/Romance...） |
| 地区分类 | 多选按钮组（China/USA/Korea/Japan...）                          |
| 年份分类 | 多选按钮组（2024/2023...）                                      |
| 标签分类 | 多选按钮组（Trending/High Rating/New Release...）               |

选择后存入 `categoryIds: [id1, id2, ...]`

#### 段三：媒体资源

| 字段          | 说明           |
| ----------- | ------------ |
| poster      | 海报图片 URL     |
| backdrop    | 背景图 URL      |
| playSources | 播放源列表（可动态增删） |

播放源按内容类型区分：

- **电影模式**：每个播放源 = `{name: "源名称", url: "m3u8播放地址"}`
- **多集模式**（电视剧/综艺/动漫/短剧）：每个播放源 = `{name: "源名称", episodes: [{season, episode, title, url, duration}, ...]}`，剧集可动态增删，每集包含 S/E 编号、标题、m3u8地址、时长

**关联数据表：** `movies`
| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 主键 |
| title | string | 标题 |
| originalTitle | string | 原文标题 |
| type | enum | movie / tv / variety / anime / short |
| overview | text | 剧情简介 |
| poster | string | 海报URL |
| backdrop | string | 背景图URL |
| releaseDate | date | 上映日期 |
| rating | decimal | 评分(0-10) |
| duration | int | 时长(分钟) |
| quality | enum | SD / HD / FHD / 4K |
| status | enum | draft / published / archived |
| viewCount | int | 播放量 |
| playSources | json | 播放源JSON数组 |
| categoryIds | json | 关联分类ID数组 |
| createdAt | datetime | 创建时间 |

### 4. 分类管理 `/categories`

**功能：**

- 按分类类型分组展示（类型/地区/年份/标签），每组一个独立卡片
- 顶部筛选：全部 / 类型 / 地区 / 年份 / 标签
- 搜索：按分类名称关键词搜索
- 新增分类：名称 + 类型 + 排序号
- 编辑分类
- 删除分类
- 每个分类项显示：名称、排序号、拖拽手柄图标、编辑/删除按钮
- 每组卡片显示该类型下的分类数量 badge

**关联数据表：** `categories`
| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 主键 |
| name | string | 分类名称（如 Action、China、2024、Trending） |
| type | enum | genre / region / year / tag |
| sortOrder | int | 排序顺序，数字越小越靠前 |

**四种分类类型：**
| type | 用途 | 示例 |
|------|------|------|
| genre | 内容类型 | Action, Comedy, Drama, Sci-Fi, Horror, Romance |
| region | 地区 | China, USA, Korea, Japan |
| year | 年份 | 2024, 2023, 2022 |
| tag | 标签 | Trending, High Rating, New Release |

### 6. 会员套餐管理 `/membership`

**功能：**

- 卡片式展示：每个套餐一张卡片，有渐变边框（按画质等级着色）
- 卡片展示字段：套餐名称、有效期天数、设备数、价格（大字突出）、画质等级图标、功能标签列表、上架/下架状态 badge
- 新增套餐 Modal：名称、价格、时长(天)、设备数、画质(下拉)、上架/下架(下拉)、功能列表(逗号分隔输入)
- 编辑套餐：回填已有数据
- 删除套餐
- 鼠标悬停显示编辑/删除按钮

**关联数据表：** `membership_plans`
| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 主键 |
| name | string | 套餐名称（如"月度VIP"） |
| price | decimal | 价格(元) |
| duration | int | 有效期(天)，0=永久免费 |
| devices | int | 同时在线设备数 |
| quality | enum | 画质：SD / HD / FHD / 4K |
| features | json | 功能列表JSON数组，如["1080P高清","无广告","离线缓存"] |
| status | int | 1=上架 0=已下架 |
| sortOrder | int | 排序 |

**示例套餐：**

- Free：免费 / SD画质 / 1设备 / 含广告
- VIP：¥29.9/月 / HD画质 / 2设备 / 无广告
- SVIP：¥49.9/月 / 4K画质 / 4设备 / 无广告+离线下载

### 7. 订单管理 `/orders`

**功能：**

- 表格列表展示：订单号、用户名、套餐名、金额、支付方式、状态（彩色badge）、创建时间
- 搜索：按订单号/用户名搜索
- 状态筛选：全部 / 已支付 / 待支付 / 已退款 / 已过期 / 已取消
- 分页
- 审核通过按钮（仅待支付状态显示）：点击确认后审核通过，为用户开通会员
- 查看详情按钮

**订单状态流转：**

```
pending(待支付) → approved → paid(已支付) → expired(已过期)
paid → refunded(已退款)
pending → cancelled(已取消)
```

**关联数据表：** `orders`
| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 订单号（如 ORD-20231120-001） |
| userId | int | 用户ID |
| username | string | 用户名（冗余） |
| planId | int | 套餐ID |
| planName | string | 套餐名称（冗余） |
| amount | decimal | 金额 |
| payment | string | 支付方式（Alipay / WeChat Pay） |
| status | enum | paid / pending / refunded / expired / cancelled |
| createdAt | datetime | 创建时间 |
| paidAt | datetime | 支付时间 |
| expiresAt | datetime | 过期时间 |

### 8. 财务管理 `/financials`

- 收入趋势柱状图（红色渐变柱体，按天/周/月聚合）
- 套餐销售分布环形饼图
- 支付方式分布环形饼图（支付宝/微信支付等）
- 收入明细列表（最近 N 条）

**统计数据：**
| 指标 | 说明 |
|------|------|
| totalRevenue | 总收入 |
| monthlyRevenue / periodRevenue | 当期收入 |
| monthlyGrowth | 环比增长(%) |
| avgOrderValue / avgOrderAmount | 平均客单价 |
| refundRate | 退款率(%) |
| newSubscribers | 新增订阅数 |
| totalOrders | 总订单数 |

admin_users ──── 管理员登录

          ├──── memberships ──── membership_plans

## 七、实现建议

1. **数据库设计**：先建 `admin_users`、`users`、`categories`、`movies`、`membership_plans`、`orders`、`system_config` 七张核心表
2. **API 开发顺序**：认证 → 影视 CRUD → 分类 CRUD → 用户管理 → 会员套餐 → 订单 → 财务统计 → 系统设置
3. **前端路由**：所有管理页面用 `/admin/xxx` 前缀，与前台用户端的 `/` 路由区分
4. **权限控制**：中间件校验 JWT token，`super_admin` 可见所有功能，`admin` 可能限制某些敏感操作
5. **支付集成**：支付宝和微信支付需要对接官方 SDK，订单状态通过回调更新
